PRD — Org Library for Teampass

Feature: Org Content Library · Surface: Desktop dashboard · Status: Draft v2

Coaches currently have no way to access org-approved content without it being manually sent to them. This feature adds a passive, browsable Org Library — accessible from the existing Content Library page via a low-hierarchy secondary button — where coaches can discover org-approved drills, videos, and documents and selectively pull items into their own team library. Org content is always read-only; coaches own nothing from the org library until they explicitly add it. The mental model is a reference shelf, not a shared folder.

PersonasCoach, Org Admin
SurfaceDesktop dashboard
DependenciesContent Library, Org/Coach role system
StageSeed · MVP scoped

User stories

As a coach, I want to browse content that my org has approved, so I can find relevant drills and documents without searching YouTube or asking the org admin directly.
As a coach, I want to add org content to my personal library in one click, so it appears alongside my own content when I publish to players.
As a coach, I want to clearly understand which content belongs to my org vs. my own team, so I always know what I can and cannot edit.
As an org admin, I want to publish approved content to a shared org library, so all coaches in my org have access to canonical program materials.

Open questions

Content updates: If the org edits or removes content after a coach has added it, does the coach's copy freeze or stay linked? Not blocking for MVP but needs a decision before launch.
Admin publish flow: How does an org admin designate content as org-level vs. team-level? This PRD assumes that flow exists — it needs its own spec.
Search: Should the org library be searchable? Deferred for now but becomes necessary past ~20 items.
Notifications: Should coaches be alerted when new org content is added? Intentionally deferred — tackle post-MVP.

Notes for Claude Design

Feed the PRD before the screenshots. Paste this full document first so Claude Design has the mental model before it sees any visuals. Leading with screenshots causes it to pattern-match to the visual and ignore intent.
Ask for one screen at a time. Start with the Org Library browse view. React, then ask for hover states, then the personal library with the org badge. Asking for the full flow in one shot collapses detail.
Be explicit about interactions. "On hover, hide the ellipsis button and show the Add to my library button" is far more useful than "make it interactive." Name the state change precisely.
Tell it what not to build. Explicitly say: no notifications, no edit capability on org content, no modal — the org library is a full-page view with a breadcrumb back.
Anchor to the token block above. Say "use only the token values in the design system block at the top of this document" — this prevents it from drifting to its own color guesses.
Best screenshots to include. Content Library page (tab pattern and layout), Upload Content page (button hierarchy and spacing), Manage Roster page (row hover and action column patterns).
